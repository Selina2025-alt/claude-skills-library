## Git 版本管理规范 (严格执行)

### 1. 存档（当我说"存档"、"提交"或"save"时）
请按顺序执行：
1. 运行 `git status` 确认当前变动。
2. 执行 `git add .`。
3. 根据改动内容生成简洁的 commit message，执行 `git commit -m "..."`。
4. **必须**读取 `CHANGELOG.md`，并在顶部追加一条记录：
   `## YYYY-MM-DD HH:mm (提交)`
   `- [改动摘要]`

### 2. 回退（当我说"回退"、"撤销"或"undo"时）
请按顺序执行：
1. **先不要执行命令**，先列出最近 3 次提交记录 (`git log --oneline -n 3`)。
2. 询问我："要回退到哪个版本？"
3. 得到我确认后，使用 `git reset --hard [commit_hash]` 执行回退。
4. 在 `CHANGELOG.md` 顶部记录：
   `## YYYY-MM-DD HH:mm (回退)`
   `- 回退到了版本：[commit_hash]`
